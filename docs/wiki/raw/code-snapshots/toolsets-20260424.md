## Public surface

- `_HERMES_CORE_TOOLS: List[str]` (line 31) — shared tool list referenced by every `hermes-<platform>` toolset (CLI + messaging platforms). Editing this list updates every gateway platform simultaneously.
- `TOOLSETS: Dict[str, dict]` (line 68) — static toolset registry. Each entry has `description: str`, `tools: List[str]` (direct tools), `includes: List[str]` (other toolsets to compose). Contains: leaf toolsets (`web`, `search`, `vision`, `image_gen`, `terminal`, `moa`, `skills`, `browser`, `cronjob`, `messaging`, `rl`, `file`, `tts`, `todo`, `memory`, `session_search`, `clarify`, `code_execution`, `delegation`, `homeassistant`); scenario toolsets (`debugging`, `safe`); and `hermes-*` toolsets (`hermes-acp`, `hermes-api-server`, `hermes-cli`, `hermes-telegram`, `hermes-discord`, `hermes-whatsapp`, `hermes-slack`, `hermes-signal`, `hermes-bluebubbles`, `hermes-homeassistant`, `hermes-email`, `hermes-mattermost`, `hermes-matrix`, `hermes-dingtalk`, `hermes-feishu`, `hermes-weixin`, `hermes-qqbot`, `hermes-wecom`, `hermes-wecom-callback`, `hermes-sms`, `hermes-webhook`, `hermes-gateway`).
- `get_toolset(name) -> Optional[Dict[str, Any]]` (line 401) — looks up name in `TOOLSETS` first, then falls through to plugin / MCP toolsets registered in `tools.registry`. Returns synthesized dict (`description`, `tools`, `includes=[]`) for plugin-registered toolsets.
- `resolve_toolset(name, visited=None) -> List[str]` (line 447) — recursively walks `includes` to flatten composed toolsets to a sorted tool-name list. Special aliases `"all"` and `"*"` resolve to the union of every registered toolset.
- `resolve_multiple_toolsets(toolset_names) -> List[str]` (line 500) — convenience wrapper that calls `resolve_toolset` for each name and returns the deduplicated sorted union.
- `get_all_toolsets()` (line 545), `get_toolset_names()` (line 570), `validate_toolset(name)` (line 593), `create_custom_toolset(name, description, tools, includes)` (line 613), `get_toolset_info(name)` (line 637) — registry inspection / mutation helpers.

- Every `hermes-<platform>` toolset (Telegram / Discord / WhatsApp / Slack / Signal / BlueBubbles / Email / Mattermost / Matrix / DingTalk / Feishu / Weixin / QQBot / WeCom / WeCom-callback / SMS / Webhook / HomeAssistant) maps `tools` directly to `_HERMES_CORE_TOOLS` — same surface for every platform.

## Invariants enforced in this module

- `resolve_toolset()` (line 447) is cycle-safe via the `visited: Set[str]` parameter — already-visited toolsets return `[]` silently (treats both diamond deps and genuine cycles uniformly).
- The `"all"` and `"*"` aliases (lines 466–471) iterate `get_toolset_names()` with a fresh `visited.copy()` per branch to prevent cross-branch contamination.

- `create_custom_toolset()` (line 613) mutates the module-global `TOOLSETS` dict directly; later calls overwrite earlier ones with the same name.
